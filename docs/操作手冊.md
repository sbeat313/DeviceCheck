# DeviceCheck 操作手冊

## 1. 專案結構
- `DeviceCheck`：主監控服務（Web API + BackgroundService）
- `DeviceCheck.Notifier`：通知接收 Console/Web 專案
- `DeviceCheck.sln`：解決方案檔

## 2. 設定方式

### 2.1 主服務 appsettings.json
```json
"DeviceCheck": {
  "BaseUrl": "http://192.168.0.58:5099",
  "CheckIntervalSeconds": 10,
  "BusyRetryDelaySeconds": 3,
  "RequestTimeoutSeconds": 10,
  "Uids": [1001, 1002, 1003]
},
"Notification": {
  "Enabled": true,
  "EndpointUrl": "http://127.0.0.1:5058/api/notifications",
  "Recipients": ["ops-team@company.local", "oncall@company.local"]
}
```

### 2.2 通知接收器 appsettings.json（已固定 Port）
```json
"Kestrel": {
  "Endpoints": {
    "Http": {
      "Url": "http://127.0.0.1:5058"
    }
  }
}
```

> 因為主服務 `Notification.EndpointUrl` 預設指向 `127.0.0.1:5058`，若兩個服務跑在同一台機器，可直接使用此預設值。

## 3. 執行順序
1. 啟動 `DeviceCheck.Notifier`（固定監聽 `http://127.0.0.1:5058`）。
2. 啟動 `DeviceCheck`。
3. DeviceCheck 開始輪詢 `/Ctrl/{uid}/RadioCheck`。
4. 當狀態在「正常(Alive)」與「異常(Busy/Dead/Unknown)」間切換時，送出通知到 Notifier。

## 4. 通知觸發規則
- **正常 -> 異常**：發送通知。
- **異常 -> 正常**：發送通知。
- 異常內部互換（例如 Busy -> Dead）不發通知。

## 5. 日誌觀察
- DeviceCheck：看監控檢查結果與通知發送狀態。
- Notifier：看收到的通知內容與對象。
