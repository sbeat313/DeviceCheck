# DeviceCheck 操作手冊

## 1. 專案結構
- `DeviceCheck`：主監控服務（Web API + BackgroundService）
- `DeviceCheck.Notifier`：通知接收 Console/Web 專案
- `DeviceCheck.sln`：解決方案檔

## 2. 設定方式

### 2.1 主設定檔 config.json
```json
{
  "DeviceCheck": {
    "BaseUrl": "http://192.168.0.58:5099",
    "CheckIntervalSeconds": 10,
    "BusyRetryDelaySeconds": 3,
    "RequestTimeoutSeconds": 10,
    "DeadConsecutiveThreshold": 3,
    "Uids": [1001, 1002, 1003],
    "UidAliases": {
      "1001": "機台A",
      "1002": "機台B"
    }
  },
  "Notification": {
    "Enabled": true,
    "EndpointUrl": "http://127.0.0.1:5058/api/notifications"
  }
}
```

- `UidAliases` 的 key 為 UID，value 為中文別名。
- 可直接修改 `config.json`，也可透過 API `PUT /api/devices/{uid}/alias` 更新。

`DeadConsecutiveThreshold` 說明：
- 連續幾次探測為 `Dead` 才正式判定為 `Dead`。
- 在達門檻前會標記為 `Unknown`。

### 2.2 appsettings.json
- 僅保留平台通用設定（例如 `Logging`、`AllowedHosts`、`NLog`）。
- 業務設定（`DeviceCheck`、`Notification`、`UidAliases`）統一放在 `config.json`。

### 2.3 通知接收器 appsettings.json（已固定 Port）
```json
"Kestrel": {
  "Endpoints": {
    "Http": {
      "Url": "http://127.0.0.1:5058"
    }
  }
}
```

## 3. Swagger 介面
- 啟動 `DeviceCheck` 後，開啟 `http://localhost:5000/swagger`（或你的實際主機與埠）。
- 可在 Swagger UI 直接呼叫心跳、狀態查詢與別名修改 API。

## 4. 執行順序
1. 啟動 `DeviceCheck.Notifier`（固定監聽 `http://127.0.0.1:5058`）。
2. 啟動 `DeviceCheck`。
3. DeviceCheck 開始輪詢 `/Ctrl/{uid}/RadioCheck`。

## 5. 通知觸發規則
- **非 Dead -> Dead（確認異常）**：發送通知。
- **Dead -> 非 Dead（異常恢復）**：發送通知。
- `Alive` / `Busy` / `Unknown` 彼此互換：不發通知。
- 通知會附帶設備別名（Alias）。

## 6. 日誌觀察
- DeviceCheck：看監控檢查結果與通知發送狀態。
- Notifier：看收到的通知內容（包含 UID 與 Alias）。
