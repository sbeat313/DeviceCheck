# DeviceCheck 操作手冊

## 1. 專案結構
- `DeviceCheck/`：主監控服務（Web API + BackgroundService）
- `DeviceCheck.Notifier/`：通知接收服務
- `DeviceCheck/docs/`：系統、API、操作文件

---

## 2. 設定

### 2.1 主設定檔：`DeviceCheck/config.json`
```json
{
  "DeviceCheck": {
    "BaseUrl": "http://192.168.0.58:5099",
    "CheckIntervalSeconds": 10,
    "RequestTimeoutSeconds": 10,
    "ProbeRetryCount": 2,
    "ProbeRetryDelaySeconds": 1,
    "DeadConsecutiveThreshold": 3,
    "Uids": [1001, 1002, 1003],
    "UidAliases": {
      "1001": "機台A",
      "1002": "機台B"
    }
  },
  "Notification": {
    "Enabled": true,
    "EndpointUrl": "http://127.0.0.1:5058/api/notifications"
  }
}
```

重點欄位：
- `CheckIntervalSeconds`：一般探測週期。
- `ProbeRetryCount`：當探測結果為 Dead/Busy 時，單輪額外重試次數。
- `ProbeRetryDelaySeconds`：Dead/Busy 重試間隔；同時作為最終結果為 Busy 時的下次檢測延遲。
- `DeadConsecutiveThreshold`：判定 Dead 的連續次數門檻。
- `UidAliases`：設備 UID 對應顯示名稱。

### 2.2 `appsettings.json`
- 保存平台通用設定（例如 `Logging`、`NLog`、`AllowedHosts`、`Kestrel`）。
- 主服務預設透過 Kestrel 監聽：`http://0.0.0.0:5000`。
- 業務設定以 `config.json` 為主。

### 2.3 Notifier 監聽位置
`DeviceCheck.Notifier/appsettings.json` 預設固定為：
- `http://127.0.0.1:5058`

---

## 3. 啟動流程
1. 啟動 `DeviceCheck.Notifier`。
2. 啟動 `DeviceCheck`。
3. DeviceCheck 開始輪詢：`/Ctrl/{uid}/RadioCheck`。

---

## 4. Swagger 與 API 測試
- 啟動主服務後，進入：`http://localhost:5000/swagger`（依實際主機/埠調整）。
- 可直接測試：
  - `POST /api/devices/{uid}/heartbeat`
  - `GET /api/devices`
  - `GET /api/devices/{uid}`
  - `PUT /api/devices/{uid}/alias`

---

## 5. 日誌觀察
- **DeviceCheck**：探測結果、狀態變更、通知發送結果。
- **Notifier**：收到通知事件的完整內容（UID、別名、狀態轉換、時間）。
