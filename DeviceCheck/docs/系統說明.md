# DeviceCheck 系統說明

本版監控與通知規則如下：

1. 背景服務定期探測每台設備，取得 `Alive` / `Busy` / `Dead` / `Unknown` 狀態。
2. `Dead` 不會因單次結果立即成立；需連續 `Dead` 達到 `DeviceCheck:DeadConsecutiveThreshold` 才判定為「確認異常」。
3. 在 `Dead` 連續次數尚未達門檻前，狀態會標記為 `Unknown`（狀態不明）。
4. 通知事件僅在「是否為確認異常 (`Dead`)」邊界改變時送出：
   - 非 `Dead` -> `Dead`：發送通知（確認異常）。
   - `Dead` -> 非 `Dead`：發送通知（異常恢復）。
5. `Alive` / `Busy` / `Unknown` 彼此之間切換不發通知。
6. 通知事件透過 `NotificationClient` 送往 `Notification.EndpointUrl`，收件者由 `Notification.Recipients` 設定。

> 目前通知語意採「已確認異常（Dead）導向」，避免過早告警。
